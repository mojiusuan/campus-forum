# 校园论坛项目 - 详细开发计划

## 一、项目概述

### 1.1 项目信息
- **项目名称：** 校园论坛平台
- **开发周期：** 4个月（MVP阶段）
- **时间投入：** 每天2小时，每周14小时，每月60小时
- **总投入：** 240小时
- **目标：** 完成MVP开发并上线运营

### 1.2 技术栈
- **前端：** React 18+ + TypeScript + Tailwind CSS
- **后端：** Node.js 18+ + Express + TypeScript
- **数据库：** PostgreSQL（Supabase）
- **部署：** Vercel（前端）+ Railway/Render（后端）
- **版本控制：** Git + GitHub

---

**任务清单：**

**数据库表结构：**

**users（用户表）**
```sql
- id: UUID (主键)
- email: VARCHAR (唯一)
- password_hash: VARCHAR
- username: VARCHAR (唯一)
- avatar_url: VARCHAR
- bio: TEXT
- is_verified: BOOLEAN (学生认证)
- created_at: TIMESTAMP
- updated_at: TIMESTAMP
```

**posts（帖子表）**
```sql
- id: UUID (主键)
- user_id: UUID (外键)
- category_id: UUID (外键)
- title: VARCHAR
- content: TEXT
- images: JSON (图片数组)
- view_count: INTEGER
- like_count: INTEGER
- comment_count: INTEGER
- is_pinned: BOOLEAN (置顶)
- is_locked: BOOLEAN (锁定)
- created_at: TIMESTAMP
- updated_at: TIMESTAMP
```

**comments（评论表）**
```sql
- id: UUID (主键)
- post_id: UUID (外键)
- user_id: UUID (外键)
- parent_id: UUID (父评论，可选)
- content: TEXT
- like_count: INTEGER
- created_at: TIMESTAMP
- updated_at: TIMESTAMP
```

**categories（分类表）**
```sql
- id: UUID (主键)
- name: VARCHAR
- slug: VARCHAR (唯一)
- description: TEXT
- icon: VARCHAR
- sort_order: INTEGER
- created_at: TIMESTAMP
```

**likes（点赞表）**
```sql
- id: UUID (主键)
- user_id: UUID (外键)
- target_type: VARCHAR (post/comment)
- target_id: UUID
- created_at: TIMESTAMP
```

**favorites（收藏表）**
```sql
- id: UUID (主键)
- user_id: UUID (外键)
- post_id: UUID (外键)
- created_at: TIMESTAMP
```

**notifications（通知表）**
```sql
- id: UUID (主键)
- user_id: UUID (外键)
- type: VARCHAR (like/comment/reply/follow)
- content: TEXT
- link: VARCHAR
- is_read: BOOLEAN
- created_at: TIMESTAMP
```

**产出：**
- 数据库设计文档
- ER图
- SQL建表语句

## 八、技术实现细节

### 8.1 后端技术细节

**认证方案：**
- JWT Token认证
- Token过期时间：7天
- Refresh Token机制（可选）

**密码加密：**
- 使用bcrypt
- Salt rounds: 10

**文件上传：**
- 使用Multer（本地）或Cloudinary（云存储）
- 图片限制：5MB
- 支持格式：jpg, png, gif

**分页：**
- 默认每页20条
- 支持自定义每页数量
- 使用cursor-based分页（可选）

**错误处理：**
- 统一错误响应格式
- 错误码定义
- 错误日志记录

### 8.2 前端技术细节

**状态管理：**
- React Query：服务器状态
- Zustand：客户端状态（用户信息、UI状态）

**路由：**
- React Router v6
- 路由守卫（保护需要登录的页面）
- 404页面

**表单处理：**
- React Hook Form
- Zod验证（可选）

**UI组件：**
- shadcn/ui（推荐）或Ant Design
- Tailwind CSS样式

**图片处理：**
- 图片懒加载
- 图片压缩（可选）
- 占位符显示
